# Static Assets — Agent Guide

This directory contains the HTML, CSS, and compiled JS for the Codex Autorunner web UI.

## Before making UI changes

Read `STYLE_GUIDE.md` in this directory. It documents design tokens, layout patterns,
responsive breakpoints, and common pitfalls learned from real iterations on the UI.

Key points:
- Use `--sp-*` tokens for all spacing, `--accent`/`--muted`/etc. for colors.
- Mobile layout uses CSS Grid (not flex-wrap heuristics) for reliable row control.
- Test at 640px and 400px widths — mobile is a first-class target.
- Panel headers are single-line and borderless. Don't add subtitle text.
- Repo card action buttons use hover-to-reveal on desktop, always-visible on mobile.

## File layout

| File | Role |
|---|---|
| `index.html` | Single-page app shell — Hub, PMA, repo views, all modals |
| `styles.css` | All CSS (single file, no preprocessor) |
| `generated/` | TypeScript build output — **do not edit directly** |
| `vendor/` | Third-party assets (fonts, libraries) |
| `assets.json` | Asset manifest (auto-generated by build) |

## Editing workflow

1. Edit TypeScript sources in `../static_src/*.ts`, not `generated/*.js`.
2. Edit `styles.css` directly for CSS changes.
3. Edit `index.html` directly for markup changes.
4. Run `pnpm run build` to compile TypeScript.
5. Run `make frontend-check` to validate (HTML lint + DOM structure check).
